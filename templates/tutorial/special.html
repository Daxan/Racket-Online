Sonderregeln

Scheme-Interpreter überprüft vor dem Auswerten den Ausdruck auf folgenden Sonderregeln und wenn keine vorhanden ist, dann wertet ihn nach den normalen Auswerteregeln aus:

  • quote: Ein Argument von quote bleibt unausgewertet! D.h. quote hat höhere Prörität.
    >(quote (+ 1 2)) => (+ 1 2)
  • if: Bei
    >(define l1 '())
    (if l1 '() (car l1))    , sollte das Ergebnis #f sein, aber es liefert: ()
  • define:
    >(define l2 '(a b c d))
    >l2 => (a b c d)
    1. Element ist das Wort define
    2. Element ist ein Symbol als Variablbezeichnung
      3. >(define 2 '(a b c d)) => Error, Variable muss Symbol sein und keine Zahl!
    3. Element ist ein Ausdruck und wird ausgewertet (-> 3.Element wird an Variable gebunden)
  • lambda:
    >(lambda (x) (+ x 1)) entspricht f(x) = x + 1
    1. Element ist das Wort lambda
    2. Element ist Parameterliste (ohne Typ)
    3. Element und ggf. weitere sind Funktionsskörper
    man kann auch für x Wert mit eingeben:
    >((lambda (x) (+ x 1))5) => f(5) =6
