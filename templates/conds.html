Bedingte Ausdrücke

Praktisches Programmieren mit funktionsorientierten Sprachen kann man als Konstruktionsaufgabe verstehen. Man konstruiert komplexere Datenstrukturen aus einfacheren und man bildet zusammengesetzte Ausdrücke aus Basisausdrücken. Dabei kommen auch bedingte Ausdrücke, oder Konditionale, vor.
In Scheme gibt es mehrere Möglichkeiten Bedingungen zu formulieren. Die einfachste Form ist das if.
Syntax:
(if test consequent alternate) oder (if test consequent)

Die Funktion if wertet zuerst das erste Argument "test" aus. Ist dies nicht #f wird das zweite Argument "consequent" ausgewertet und als Ergebnis zurückgegeben. Das dritte Argument "alternate" ist der optionale "else Teil".
(define x 42)
(if (= x 42) "richtig" "falsch" ) => "richtig"

Die Funktion cond kann verwendet werden, anstatt mehrere if-else Bedingungen zu schachteln. Die Argumente müssen jeweils eine Bedingung und einen Ausdruck, der ausgewertet werden soll, wenn die Bedingung #t ist, enthalten. Als letztes Argument ist ein else Teil möglich, der ausgewertet wird, wenn keine Bedingung #t lieferte.
Wir erläutern dies an folgendem Beispiel.
(define x 44)
(cond ((< x 42) "kleiner") ((> x 42) "größer") (else "richtig")) => "größer"
